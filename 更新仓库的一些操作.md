# 🚀 Git 核心工作流总结 (SSH 模式)

这是我使用 Git Bash 和 VS Code 进行版本控制的两个核心工作流。

---

## 🛠️ 工作流 A：创建和发布一个全新的仓库 (仅运行一次)

当开始一个全新的学习项目（例如 `Python-Projects`）时，使用此流程。

| 步骤 | 目的 | 命令（按顺序执行） | 备注 |
| :---: | :--- | :--- | :--- |
| **1.** | **创建目录** | `mkdir New-Repo-Name` | 在 Git Bash 中创建文件夹。 |
| **2.** | **进入目录** | `cd New-Repo-Name` | |
| **3.** | **初始化仓库** | `git init` | **【核心 1】**：创建本地 `.git` 仓库。 |
| **4.** | **创建/编辑文件** | `code .` $\rightarrow$ 在 VS Code 中创建/编辑文件。 | |
| **5.** | **首次提交** | `git add .` $\rightarrow$ `git commit -m "Initial commit"` | **【核心 2 & 3】**：打包并记录第一个版本。 |
| **6.** | **GitHub 创建** | **浏览器操作：** 在 GitHub 创建同名空仓库，获取 **SSH 链接**。 | `git@github.com:user/repo.git` |
| **7.** | **关联远程仓库** | `git remote add origin [SSH 链接]` | **【核心 4】**：指定远程目的地。 |
| **8.** | **首次推送** | `git push -u origin master` (或 `main`) | **【核心 5】**：同步到 GitHub 并设置跟踪。 |

---

## 💻 工作流 B：日常更新已创建的仓库 (每天重复使用)

在 `git-learning` 或 `Machine-Learning-Notes` 等已关联仓库中继续工作时，使用此简洁流程。

| 步骤 | 目的 | 命令（按顺序执行） | 备注 |
| :---: | :--- | :--- | :--- |
| **1.** | **进入目录** | `cd /e/Repo-Name` | |
| **2.** | **开始编辑** | `code .` | 启动 VS Code 进行修改/编写笔记。 |
| **3. a** | **暂存/打包** | `git add .` | 打包所有修改。 |
| **3. b** | **提交/记录** | `git commit -m "本次更新的简短描述"` | 确保描述有意义。 |
| **4.** | **同步到云端** | `git push` | **最简单的命令**，一键推送到 GitHub。 |
